# P1 真实完成度审计与精确补全方案

**审计时间**: 2026-04-10  
**审计方式**: 逐页浏览器实测 + 代码审查  
**判定标准**: 用户登录后能看到什么、能操作什么、数据是否真实

## 一、诚实评估：用户实际看到了什么

### 1. 告警中心（用户感知：60% 可用）

**已实现且有真实数据的：**

- ✅ 告警事件列表：有真实告警数据（7 条触发中，1 条已确认，12 条已恢复）
- ✅ 告警确认/关闭：可操作，状态流转正常
- ✅ 批量确认/关闭：可操作
- ✅ 自动刷新：15 秒轮询
- ✅ 收敛分组 Tab：可切换，有数据
- ✅ 时间轴 Drawer：可打开，展示触发/确认/恢复节点
- ✅ 根因分析 Drawer：可打开，展示证据链和置信度
- ✅ 评论功能：可添加评论

**有代码但用户看到的是空壳/报错的：**

- ⚠️ 收敛分组的「首触发」「末触发」「时长」列大概率显示空（数据库时间格式可能不匹配前端解析）
- ⚠️ 拓扑视图：点击后大概率显示「该告警未关联服务树」（绝大多数告警没有 service_tree_id）
- ⚠️ 告警抑制：后端逻辑存在，但前端没有任何展示 suppressed 状态的 UI
- ⚠️ 告警升级：后端定时器在跑，但前端没有升级状态的展示
- ⚠️ 指派功能：后端 API 存在，但前端没有指派按钮/UI

**完全缺失的：**

- ❌ 告警通知渠道实际送达（邮件/webhook 需要配置才能验证）
- ❌ 告警与工单的前端关联跳转（ticket_id 只显示数字，没有链接）
- ❌ 告警事件详情页（只有列表，没有独立详情页展示完整信息）

### 2. 监控大盘（用户感知：50% 可用）

**已实现且有真实数据的：**

- ✅ Agent 总数/在线/离线统计卡片：有真实数据
- ✅ Agent 实时列表：有真实数据，支持搜索和分页
- ✅ Agent 趋势 Drawer：可查看 CPU/内存/磁盘趋势图
- ✅ Golden Signals 四大信号卡片：有数据
- ✅ SLO 达标/未达标标签：有显示
- ✅ 维度拆分表格：有数据
- ✅ SLO 配置弹窗：可打开，可修改

**有代码但用户看到的是空壳的：**

- ⚠️ 异常检测表格：显示「未发现异常指标」（需要至少 10 个采样点才能计算基线）
- ⚠️ 容量预测表格：显示「暂无预测数据」（需要至少 5 天的历史数据才能做线性回归）
- ⚠️ 热点 Top：如果没有 Agent 超过 80% 阈值就全空
- ⚠️ 服务树聚合/负责人聚合：如果资产没有关联服务树和负责人就全空
- ⚠️ 延迟指标：虽然加了 RTT 计算，但 Agent 心跳的 timestamp 字段是 proto 定义的，如果 Agent 端没有更新 proto 发送 timestamp，RTT 永远是 0

**完全缺失的：**

- ❌ 没有历史趋势对比（只能看当前窗口，不能看昨天 vs 今天）
- ❌ 没有自定义告警阈值在大盘上的可视化标线
- ❌ Golden Signals 没有趋势图（只有数字卡片，没有折线图）

### 3. 巡检中心（用户感知：40% 可用）

**已实现且有真实数据的：**

- ✅ 模板创建/编辑表单：必填校验、任务模板绑定
- ✅ 计划创建/编辑表单：Cron 表达式
- ✅ 立即执行：可触发（需要有在线 Agent）
- ✅ 执行记录列表：有数据
- ✅ 执行后轮询终态
- ✅ 报告 Drawer：结构化展示（概览 + 主机结果表格）
- ✅ 导出 JSON/CSV
- ✅ 趋势 Drawer

**有代码但用户看到的是空壳的：**

- ⚠️ 修复任务字段：表单有了，但修复任务的实际触发依赖 SyncRecordStatus，而 SyncRecordStatus 依赖 gRPC Agent 回调，如果 Agent 不在线则不触发
- ⚠️ 记录对比（Diff）：UI 有了，但如果只有 1 条执行记录无法对比；且对比结果依赖 report_json 里有 host_results，如果执行没完成就是空
- ⚠️ Cron 调度器：代码在跑，但如果没有在线 Agent，定时执行会一直失败

**完全缺失的：**

- ❌ 没有巡检模板删除功能（只能创建和编辑）
- ❌ 没有巡检计划删除功能
- ❌ 没有巡检计划启用/禁用切换
- ❌ 没有巡检执行记录的状态筛选（全部/成功/失败/运行中）
- ❌ 没有巡检模板的主机列表展示（default_hosts 是 JSON，前端列表里没展示）
- ❌ 巡检报告没有失败主机标红置顶

### 4. 跨模块联动（用户感知：10% 可用）

- ⚠️ 告警→工单：后端代码存在（createAlertTicket），但前端告警列表里看不到关联的工单链接
- ⚠️ 工单关闭→告警自动恢复：后端代码存在（autoResolveLinkedAlerts），但需要真实走完工单流程才能验证
- ⚠️ 巡检→告警→工单：巡检失败会创建告警事件，但告警事件里看不到是从哪个巡检记录来的
- ⚠️ 变更风险评估：后端 API 存在，但前端没有任何入口调用它

## 四、总结

**当前 P1 的真实状态**：后端 API 覆盖率约 85%，前端页面覆盖率约 50%，端到端可用率约 40%。

**核心问题**：

1. 很多功能只有后端 API，前端没有入口或按钮
2. 有些功能有 UI 但数据链没通（拓扑、异常检测、容量预测）
3. 基础 CRUD 不完整（巡检没有删除、没有启用/禁用）
4. 跨模块跳转链接缺失（告警→工单、巡检→告警）
5. 状态展示不全面（suppressed、escalated 前端没体现）

**修复后的效果**：用户登录后每个页面都能看到真实数据和完整操作，不再有空壳感。
