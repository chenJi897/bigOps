# BigOps AI 长周期高难模块持续开发方案

**适用项目**: BigOps（backend Go + frontend Vue3）  
**目标**: 支撑 AI 长时间连续开发高难模块，稳定产出“可运行功能 + 自动化测试 + 浏览器实测 + 可持续迭代代码”。  
**执行方式**: 不设固定时长，按里程碑推进；每个里程碑必须通过质量闸门后再进入下一阶段。

---

## 一、硬性约束（每个里程碑必须满足）

1. 必须同时包含前端和后端改动（除纯文档轮次外）。
2. 必须新增或更新自动化测试（Go test 或脚本回归），且覆盖正常路径 + 异常路径。
3. 必须执行并通过：
  - 后端编译：`go build ./cmd/core/main.go`
  - 前端类型检查：`npx vue-tsc -b --noEmit`
4. 必须完成浏览器实测（至少 8 条检查项，含跨模块链路与异常路径）。
5. 必须提交代码并推送（如用户未禁止）。
6. 每阶段结束必须输出阶段结果、风险与下一阶段入口条件，不得“做完再说”。

---

## 二、长周期执行节奏（固定模板）

## Phase 0：范围冻结与架构边界

- 选择 **1 个高难主模块 + 1 个配套子模块**，避免并行开太多战线。
- 输出：
  - 本阶段改动文件清单（预计）
  - 验收标准（可量化）
  - 风险点（最多 5 条）

**本项目建议高难主模块池（按优先级）**：

1. 任务中心：高危任务审批与执行门禁（策略化 + 可审计）
2. 告警中心：SLA/静默/升级自动化闭环（可回放、可追责）
3. 巡检中心：计划冲突治理 + 重试策略 + 健康评分趋势
4. 监控中心：Golden Signals 可解释性与异常归因
5. 工程交付：容器化、回归脚本、环境一致性

---

## Phase 1：核心能力实现

### 后端改动路径

`model -> repository -> service -> handler -> router`

必须覆盖（高难模块最低要求）：

- 参数校验（返回明确错误信息）
- 审计日志（关键操作写 activity 或 audit）
- 状态流转正确（禁止非法状态跳转）
- 并发/重复提交防护（幂等键、状态锁或唯一约束）
- 可观测性（关键日志字段 + 失败原因可检索）

### 前端改动路径

`api/index.ts -> 页面组件 -> 状态展示/交互 -> 错误提示`

必须覆盖（高难模块最低要求）：

- 入口可达（按钮/菜单/跳转）
- 正常路径可走通
- 错误路径有明确提示（不是只弹通用失败）
- 关键状态可视化（审批中/已超时/冲突中/重试中）
- 复杂操作防误触（确认弹窗、禁用态、二次确认）

## Phase 2：自动化测试与回归

### 后端

- 至少覆盖 4 类场景：
  - 正常场景
  - 非法输入
  - 状态冲突/并发场景
  - 边界值场景（空值、极值、超时）

建议命令（按改动范围选）：

```bash
cd backend
go test ./internal/service -run <TestName> -v
go test ./internal/handler -run <TestName> -v
```

### 回归脚本（建议每阶段至少跑一次）

```bash
BASE_URL="http://127.0.0.1:8080" \
USERNAME="admin" \
PASSWORD="<你的密码>" \
HOST_IPS="127.0.0.1" \
MODE=quick \
./backend/scripts/task_center_smoke.sh
```

## Phase 3：编译验证（必须）

```bash
cd backend
go build ./cmd/core/main.go

cd ../frontend
npx vue-tsc -b --noEmit
npm run build
```

若失败：必须先修复再进入下一阶段，不允许带错进入浏览器实测或提交。

---

## Phase 4：浏览器实测（必须）

使用 Cursor 浏览器或手工浏览器，按以下清单逐项勾选：

- 页面入口可达（菜单/按钮可进入目标页面）
- 正常流程成功（创建/编辑/执行/查看）
- 异常流程拦截（空值、非法值、状态不允许）
- 数据回写正确（列表与详情一致）
- 跨模块跳转正常（如告警->工单、任务->执行）
- 并发场景可控（重复点击/重复提交不产生脏数据）
- 状态刷新一致（页面刷新后状态不丢）
- 权限边界正确（无权限用户不可执行高危操作）

必须附带：

- 实测页面 URL
- 关键截图（至少 3 张）
- 失败项及原因

## Phase 5：提交与交付

提交信息格式建议：

`feat(scope): 本阶段主目标 + test`

提交说明必须包含：

1. 改动文件列表
2. 测试命令与结果
3. 浏览器实测结果

## 三、本项目“长周期高难模块”推荐路线（6大里程碑）

## Milestone-1（任务审批门禁中台）

- 目标：高危任务执行前必须审批，支持审批意见与审批记录追溯
- 核心点：风险策略化（命令模式 + 执行用户 + 目标范围）+ 审批状态机
- 验收：未审批执行返回 409；审批通过可执行；全链路审计可查

## Milestone-2（告警自动闭环）

- 目标：SLA 超时自动升级 + 静默过期自动失效 + 责任人催办
- 核心点：告警状态与任务联动（告警 -> 任务 -> 执行结果回写）
- 验收：超时活动自动生成，前端可见升级轨迹，静默规则按期失效

## Milestone-3（巡检策略引擎）

- 目标：计划冲突阻断 + 失败自动重试 + 健康评分趋势
- 核心点：冲突检测前置、重试上限策略、评分口径统一
- 验收：冲突计划阻断；失败按策略重试；周/月趋势可视化

## Milestone-4（监控可解释性与归因）

- 目标：Golden Signals 增加覆盖率/可信度/异常归因
- 核心点：实例/服务/接口维度真正区分，展示数据口径与来源
- 验收：维度结果有差异且可解释；可信度可追踪

## Milestone-5（工程化与环境一致性）

- 目标：容器化启动 + 配置标准化 + 回归脚本化
- 核心点：`docker-compose` 一键拉起、环境变量规范、最小演示链路
- 验收：`docker-compose up` 后可登录并走通“任务创建->执行->查看”

## Milestone-6（稳定性与性能强化）

- 目标：补齐高频慢接口、长列表分页、关键页面加载优化
- 核心点：后端索引/查询优化、前端懒加载与缓存策略
- 验收：核心列表接口耗时下降，首屏可用时间稳定

## 四、给 AI 的长周期固定指令模板（可直接复制）

```text
从现在开始执行 BigOps 长周期高难模块开发，严格按以下流程：
1) 先冻结本阶段范围（1个高难主模块+1个配套子模块），写出验收标准；
2) 按 model/repo/service/handler/router + api/view 实现；
3) 必须新增或更新测试，并覆盖正常、异常、冲突、边界场景；
4) 必须执行 go build + vue-tsc + npm run build；
5) 必须做浏览器实测（>=8项清单）；
6) 修完所有报错后提交并推送；
7) 最后输出：改动文件、测试结果、实测结果、风险清单、下一阶段建议。
禁止跳过测试和浏览器实测。
```
