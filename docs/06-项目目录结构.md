## 二、后端目录说明

### 2.1 cmd/ - 应用程序入口

每个模块有独立的入口文件，负责：
- 初始化配置
- 初始化数据库连接
- 初始化路由
- 启动HTTP/gRPC服务

### 2.2 internal/ - 内部代码

#### model/ - 数据模型
- 定义数据库表结构
- GORM模型定义
- 模型关联关系

#### service/ - 业务逻辑层
- 实现业务逻辑
- 调用repository层
- 处理业务规则

#### handler/ - HTTP处理器
- 接收HTTP请求
- 参数验证
- 调用service层
- 返回响应

#### middleware/ - 中间件
- 认证中间件
- 权限中间件
- 日志中间件
- 限流中间件

#### repository/ - 数据访问层
- 数据库操作
- 缓存操作
- 数据查询封装

#### pkg/ - 内部公共包
- 数据库连接
- 配置管理
- 日志管理
- 工具函数

### 2.3 api/ - API定义

#### http/ - HTTP API
- 路由定义
- 路由分组
- 中间件配置

#### grpc/ - gRPC API
- Proto文件定义
- gRPC服务实现

### 2.4 config/ - 配置文件

- 数据库配置
- Redis配置
- JWT配置
- 日志配置
- 多环境配置

### 2.5 migrations/ - 数据库迁移

- SQL迁移脚本
- 版本管理
- 数据初始化

## 六、目录设计原则

### 6.1 模块化

- 按功能模块组织代码
- 模块间低耦合
- 模块内高内聚

### 6.2 分层架构

- 表现层（Handler）
- 业务层（Service）
- 数据层（Repository）
- 模型层（Model）

### 6.3 可扩展性

- 易于添加新模块
- 易于添加新功能
- 易于维护和重构

### 6.4 规范统一

- 命名规范统一
- 目录结构统一
- 代码风格统一

## 七、总结

本目录结构设计遵循Go和Vue3的最佳实践，采用分层架构和模块化设计，既保证了代码的可维护性，又提供了良好的扩展性。通过清晰的目录组织，开发人员可以快速定位代码，提高开发效率。
