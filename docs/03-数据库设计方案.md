# BigOps 大运维平台 - 数据库设计方案

**版本**: v1.0
**日期**: 2026-03-17
**状态**: 已定稿

## 一、数据库选型

### 1.1 关系型数据库：MySQL 8.0+

**选型理由**:
- 成熟稳定，社区活跃
- 性能优秀，满足中小团队需求
- 运维工具丰富
- 团队熟悉度高

**配置建议**:
```ini
# 连接配置
max_connections = 500
max_connect_errors = 100

# 缓冲池配置
innodb_buffer_pool_size = 2G
innodb_log_file_size = 512M

# 字符集配置
character_set_server = utf8mb4
collation_server = utf8mb4_unicode_ci

# 时区配置
default-time-zone = '+08:00'
```

### 1.2 缓存数据库：Redis 7.0+

**用途**:
- JWT Token存储
- 会话管理
- 任务队列（Asynq）
- 热点数据缓存
- 分布式锁

## 二、数据库设计原则

### 2.1 命名规范

- **表名**: 小写字母，下划线分隔，复数形式（如：users, roles）
- **字段名**: 小写字母，下划线分隔（如：user_id, created_at）
- **索引名**: idx_表名_字段名（如：idx_users_username）
- **外键名**: fk_表名_字段名（如：fk_users_role_id）

### 2.2 通用字段

所有业务表包含以下审计字段：
- `id`: 主键，BIGINT UNSIGNED AUTO_INCREMENT
- `created_at`: 创建时间，DATETIME
- `updated_at`: 更新时间，DATETIME
- `deleted_at`: 删除时间，DATETIME（软删除）

### 2.3 设计规范

- 使用InnoDB存储引擎
- 主键使用自增ID
- 外键使用索引
- 合理使用索引（避免过度索引）
- 敏感字段加密存储
- 大文本字段使用TEXT类型
- 时间字段使用DATETIME类型

```sql

  `id` bigint unsigned NOT NULL AUTO_INCREMENT COMMENT '用户ID',
  `username` varchar(50) NOT NULL COMMENT '用户名',
  `password` varchar(100) NOT NULL COMMENT '密码（bcrypt加密）',

  `last_login_ip` varchar(50) DEFAULT NULL COMMENT '最后登录IP',
  `created_at` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `deleted_at` datetime DEFAULT NULL COMMENT '删除时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_username` (`username`),
  KEY `idx_email` (`email`),
  KEY `idx_status` (`status`),
  KEY `idx_deleted_at` (`deleted_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户表';
```

## 四、索引设计

### 4.1 索引原则

- 主键自动创建聚簇索引
- 外键字段创建索引
- 频繁查询的字段创建索引
- 频繁排序的字段创建索引
- 唯一约束字段创建唯一索引
- 组合索引遵循最左前缀原则

### 4.2 索引优化建议

- 避免在低选择性字段创建索引
- 控制单表索引数量（不超过5个）
- 定期分析索引使用情况
- 删除冗余索引

## 五、数据安全

### 5.1 敏感数据加密

- 用户密码：Bcrypt加密
- 数据库密码：AES加密
- API密钥：AES加密

### 5.2 备份策略

- 每日全量备份（凌晨2点）
- binlog增量备份（实时）
- 备份保留30天
- 定期恢复演练

### 5.3 权限控制

- 应用账号：只读/读写权限分离
- 管理账号：严格控制使用
- 禁止root账号直接访问

## 六、性能优化

### 6.1 查询优化

- 避免SELECT *
- 使用LIMIT限制返回行数
- 合理使用JOIN
- 避免子查询，使用JOIN替代

### 6.2 表优化

- 定期执行OPTIMIZE TABLE
- 定期分析表统计信息
- 大表考虑分区或分表

### 6.3 连接池配置

```go
// GORM连接池配置
db.DB().SetMaxOpenConns(100)  // 最大打开连接数
db.DB().SetMaxIdleConns(10)   // 最大空闲连接数
db.DB().SetConnMaxLifetime(time.Hour) // 连接最大生命周期
```

## 七、数据迁移

## 八、总结

本数据库设计方案遵循规范化设计原则，合理使用索引，注重数据安全和性能优化。通过软删除、审计字段等设计，保证了数据的可追溯性。后续可根据实际使用情况，进行针对性优化。
