## 一、部署架构概述

### 1.1 架构设计原则

- **混合架构**: 核心模块单体部署，独立模块容器化部署
- **容器化**: 使用Docker容器化部署，便于管理和扩展
- **服务隔离**: 独立模块单独部署，互不影响
- **高可用**: 关键服务支持多实例部署
- **易维护**: 统一的部署和管理方式

### 1.2 部署模式

#### 单机部署（推荐用于开发/测试环境）
- 所有服务部署在单台服务器
- 使用Docker Compose编排
- 资源需求：4核8G内存，100G磁盘

#### 分布式部署（推荐用于生产环境）
- 核心服务、独立模块、数据库分别部署
- 支持服务横向扩展
- 资源需求：根据实际负载配置

## 四、Dockerfile配置

### 4.1 Dockerfile.core（核心模块）

```dockerfile
# 构建阶段
FROM golang:1.21-alpine AS builder

WORKDIR /build

# 安装依赖
RUN apk add --no-cache git make

# 复制go mod文件
COPY backend/go.mod backend/go.sum ./
RUN go mod download

# 复制源码
COPY backend/ ./

# 编译
RUN CGO_ENABLED=0 GOOS=linux go build -a -installsuffix cgo -o bigops-core ./cmd/core

# 运行阶段
FROM alpine:latest

WORKDIR /app

# 安装必要工具
RUN apk --no-cache add ca-certificates tzdata curl

# 设置时区
ENV TZ=Asia/Shanghai

# 复制二进制文件
COPY --from=builder /build/bigops-core .

# 复制配置文件
COPY backend/config/config.yaml ./config/

# 暴露端口
EXPOSE 8080

# 健康检查
HEALTHCHECK --interval=30s --timeout=10s --retries=3 \
  CMD curl -f http://localhost:8080/health || exit 1

# 启动服务
CMD ["./bigops-core"]
```

## 五、Nginx配置

### 5.1 nginx.conf

```nginx
user nginx;
worker_processes auto;
error_log /var/log/nginx/error.log warn;
pid /var/run/nginx.pid;

events {
    worker_connections 1024;
    use epoll;
}

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                    '$status $body_bytes_sent "$http_referer" '
                    '"$http_user_agent" "$http_x_forwarded_for"';

    access_log /var/log/nginx/access.log main;

    sendfile on;
    tcp_nopush on;
    tcp_nodelay on;
    keepalive_timeout 65;
    types_hash_max_size 2048;
    client_max_body_size 100M;

    gzip on;
    gzip_vary on;
    gzip_proxied any;
    gzip_comp_level 6;
    gzip_types text/plain text/css text/xml text/javascript
               application/json application/javascript application/xml+rss
               application/rss+xml font/truetype font/opentype
               application/vnd.ms-fontobject image/svg+xml;

    include /etc/nginx/conf.d/*.conf;
}
```

## 六、部署流程

### 6.1 环境准备

```bash
# 安装Docker
curl -fsSL https://get.docker.com | bash

# 安装Docker Compose
curl -L "https://github.com/docker/compose/releases/download/v2.20.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
chmod +x /usr/local/bin/docker-compose

# 验证安装
docker --version
docker-compose --version
```

### 6.2 部署步骤

```bash
# 1. 克隆代码
git clone https://github.com/your-org/bigops.git
cd bigops

# 2. 构建前端
cd frontend
npm install
npm run build
cd ..

# 3. 配置环境变量
cd deploy
cp .env.example .env
vim .env  # 修改配置

# 4. 启动服务
docker-compose up -d

# 5. 查看日志
docker-compose logs -f

# 6. 检查服务状态
docker-compose ps
```

### 6.3 初始化数据

```bash
# 进入核心模块容器
docker exec -it bigops-core sh

# 执行数据库迁移
./bigops-core migrate

# 创建管理员账号
./bigops-core admin create --username admin --password admin123
```

## 七、运维管理

### 7.1 服务管理

```bash
# 启动所有服务
docker-compose up -d

# 停止所有服务
docker-compose down

# 重启服务
docker-compose restart [service_name]

# 查看日志
docker-compose logs -f [service_name]

# 进入容器
docker exec -it [container_name] sh
```

### 7.2 数据备份

```bash
# 备份MySQL数据
docker exec bigops-mysql mysqldump -u bigops -pbigops123 bigops > backup_$(date +%Y%m%d).sql

# 备份Redis数据
docker exec bigops-redis redis-cli SAVE
docker cp bigops-redis:/data/dump.rdb ./backup/redis_$(date +%Y%m%d).rdb
```

### 7.3 监控告警

```bash
# 查看容器资源使用
docker stats

# 查看容器健康状态
docker ps --format "table {{.Names}}\t{{.Status}}"
```

## 八、总结

本部署架构方案采用Docker容器化部署，通过Docker Compose编排管理，实现了服务的快速部署和便捷管理。混合架构设计既保证了核心模块的简单性，又为独立模块的扩展提供了灵活性。
