# 首页仪表盘改版设计

**日期**: 2026-03-27  
**状态**: 已确认设计  
**目标模块**: 首页 Dashboard

---

## 一、目标

把当前单一的“平台总览”首页改成：

- 上半区：**个人工作台**
- 下半区：**按权限显示的平台总览**

要求：

1. 普通用户优先看到和自己相关的待办、申请、任务、告警
2. 普通用户不能看到超出自身权限的数据卡片
3. 管理员保留完整的平台全局视角
4. 首页视觉和交互要更像“工作台”，而不是单纯统计卡片堆叠

---

## 二、信息架构

### 2.1 双层结构

首页分两层：

1. **个人工作台**
   - 我的待办工单
   - 我的申请工单
   - 我负责的资产
   - 我相关的告警
   - 我最近执行的任务
   - 我最近触发的流水线

2. **平台总览**
   - 仅显示当前用户有权限访问的模块
   - 管理员显示完整平台总览
   - 普通用户不显示自己没有权限进入的模块卡片

### 2.2 权限规则

页面展示遵循两层限制：

1. **模块权限**
   - 通过 `permissionStore.menus` 判断当前用户是否拥有模块入口
   - 无权限模块不显示卡片、不显示快捷入口

2. **数据范围**
   - 个人工作台只看“与我有关”的数据
   - 平台总览：
     - 管理员显示平台全局
     - 普通用户显示“可访问模块的摘要”，但不强求都是个人范围

---

## 三、页面结构

### 3.1 头部欢迎区

- 标题：根据用户角色动态变化
  - 管理员：`平台总览`
  - 普通用户：`工作台`
- 副标题：
  - 管理员：强调整体态势
  - 普通用户：强调待处理事项和个人动作
- 快捷入口区：
  - 发起工单
  - 我的待办
  - 告警事件
  - 运行记录
  - 任务执行
  - 按权限显示

### 3.2 个人工作台区

建议做成 6 张卡片：

- 我的待办
- 我的申请
- 我的相关告警
- 我的任务执行
- 我的流水线运行
- 我的资产/负责资产

普通用户：首页第一屏重点显示这一区。

管理员：首页也显示这一区，但权重低于平台总览。

### 3.3 平台总览区

按权限显示以下模块卡片：

- CMDB
  - 主机资产
  - 云账号
  - 服务树
- 监控
  - 在线 Agent
  - 触发中告警
- 工单
  - 打开中的工单
  - 全部工单
- 任务中心
  - 最近任务执行
- CI/CD
  - 流水线
  - 最近运行
- 系统管理
  - 用户数

### 3.4 趋势与排行区

保留两类可视化：

- 资产来源分布
- 服务树 Top 10

但它们也要按权限显示：

- 没有 CMDB 权限就不显示

## 四、后端接口设计

### 4.1 新增接口

新增一个“个人工作台摘要”接口：

- `GET /api/v1/dashboard/personal`

返回字段建议：

- `my_pending_tickets`
- `my_created_tickets`
- `my_related_alerts`
- `my_task_executions`
- `my_pipeline_runs`
- `my_assets`

### 4.2 继续复用接口

平台总览继续复用：

- `GET /api/v1/stats/summary`
- `GET /api/v1/stats/asset-distribution`

监控摘要可继续复用：

- `GET /api/v1/monitor/summary`

### 4.3 权限控制方式

后端不替前端决定模块显隐。

前端通过菜单树决定显示哪些模块，
后端只负责返回：

- 个人摘要
- 平台统计
- 监控摘要

## 五、前端实现策略

### 5.1 页面层

修改：

- `frontend/src/views/Dashboard.vue`

复用：

- `permissionStore`
- `userStore`
- `statsApi`
- `monitorApi`
- 新增 `dashboardApi.personal()`

### 5.2 组件策略

不拆很多新组件，优先在现有页面内完成：

- 欢迎区
- 快捷入口
- 个人卡片区
- 平台卡片区
- 分布区

### 5.3 显隐策略

封装一组 computed：

- `canViewCMDB`
- `canViewMonitor`
- `canViewTicket`
- `canViewTask`
- `canViewCICD`
- `canViewSystem`

它们基于 `permissionStore.menus` 解析。

---

## 六、验收标准

1. 管理员打开首页，能看到：
   - 个人工作台
   - 完整平台总览
2. 普通用户打开首页，只看到：
   - 自己相关的工作台内容
   - 自己有权限模块的总览卡片
3. 没有权限的模块：
   - 不显示卡片
   - 不显示快捷入口
4. 浏览器验收：
   - Playwright 通过
   - Chrome DevTools / CDP 通过
   - 无 console error
   - 无 failed response

---

## 七、结论

首页不再只做“平台统计墙”，而是改成：

```text
欢迎区 / 快捷入口
-> 个人工作台
-> 平台总览
-> 分布与排行
```

管理员看到全局，普通用户看到自己能做什么、当前该做什么。  
这才符合运维平台首页的角色。
