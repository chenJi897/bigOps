# BigOps 各模块缺陷分析与优化方案

> 基于代码实际审查 | 2026-03-31

---

## 一、Module 01 底座（用户/角色/权限）

### 当前缺陷

| 问题 | 严重度 | 代码位置 |
|------|--------|---------|
| 无 MFA/2FA | 高 | `auth_handler.go` 仅支持密码登录 |
| 无 OAuth/LDAP/OIDC 对接 | 高 | 无第三方身份源集成 |
| 密码复杂度偏弱 | 中 | `auth_service.go:34` 仅 min=6 |
| 禁用账号返回"已被禁用"泄露用户存在 | 中 | `auth_handler.go:98-103` |
| 部门层级未做数据隔离 | 中 | A 部门用户能看到 B 部门资产 |
| 角色权限无矩阵视图 | 低 | 前端只能逐个角色查看菜单权限 |
| 审计日志可被篡改 | 低 | 无不可变存储/签名机制 |
| 无用户邀请机制 | 低 | 只能管理员手动创建 |

### 优化建议

1. **TOTP 二次验证** — 对接 Google Authenticator，登录后输 6 位码
2. **LDAP/OAuth 集成** — 企业场景 90% 有 AD 域或飞书/钉钉 SSO
3. **权限矩阵视图** — 一张表看所有角色 × 所有菜单的勾选情况
4. **部门数据隔离** — 资产、工单等按部门/服务树自动过滤
5. **登录安全增强** — 异地登录告警、设备指纹、会话管理

## 二、Module 02 CMDB（资产/服务树/云同步）

### 当前缺陷

| 问题 | 严重度 | 代码位置 |
|------|--------|---------|
| 无 Excel 导入导出 | 高 | 资产只能逐条创建 |
| 无批量操作 | 高 | 无法批量变更服务树/标签/负责人 |
| 资产关系无拓扑 | 高 | 无 web→db 的依赖关系 |
| OwnerIDs 存 JSON 字符串 | 中 | `model/asset.go:27` 无外键校验 |
| 无并发更新保护 | 中 | 两人同时编辑最后写入的赢 |
| 云同步无进度展示 | 中 | 只显示"同步中"，不显示 50/200 |
| 无费用管理 | 中 | 缺 cost_per_month 字段 |
| 无到期预警 | 低 | `expired_at` 字段存在但未用 |
| Tags 无受控词表 | 低 | 可以填任意字符串 |

### 优化建议

1. **Excel 导入导出** — 资产列表支持 CSV/Excel 下载和批量导入
2. **批量操作工具栏** — 勾选多条后可批量修改归属/标签/状态
3. **资产拓扑图** — 基于服务树展示应用 → 中间件 → 数据库依赖链
4. **乐观锁** — 加 version 字段防止并发覆盖
5. **到期预警** — 云主机/域名/证书到期前 N 天自动创建工单
6. **费用看板** — 按服务树/部门聚合月度云资源开销

## 三、Module 03 工单系统

### 当前缺陷

| 问题 | 严重度 | 代码位置 |
|------|--------|---------|
| 无 SLA 机制 | 高 | 无响应时间/处理时限/超时升级 |
| 审批无并行/加签/委托 | 高 | `approval_handler.go` 仅 approve/reject |
| 无批量审批 | 中 | 10 条同类工单要点 10 次通过 |
| 表单无条件渲染 | 中 | 模板字段全部展示，不能按类型切换 |
| 工单无关联/合并 | 中 | 重复工单无法合并去重 |
| 评论无内部/外部区分 | 低 | 所有评论对申请人可见 |
| 状态流转无强校验 | 低 | `ticket_service.go:17-23` 定义了但未强制 |

### 优化建议

1. **SLA 引擎** — 按模板配置响应时限（如 P0 工单 15 分钟内响应）
   - 超时自动升级到上级
   - 工单列表显示"剩余 XX 分钟"倒计时
2. **审批增强** — 并行审批（多人会签）、加签、转签、委托
3. **批量审批** — 勾选多条 → 一键批量通过/拒绝
4. **动态表单** — 模板 form_schema 支持条件渲染
5. **工单度量** — 平均处理时长、超时率、各模板使用量

## 四、Module 04 任务执行中心

### 当前缺陷

| 问题 | 严重度 | 代码位置 |
|------|--------|---------|
| 无定时任务/Cron | 高 | CreateTaskRequest 无 schedule 字段 |
| 无任务依赖编排 | 高 | 无 A 完成后才执行 B |
| 脚本无语法高亮 | 中 | 前端 textarea 原始编辑 |
| 无密钥注入 | 中 | 脚本中密码只能写明文 |
| 无主机组概念 | 中 | 每次执行手动输入 IP 列表 |
| 无 dry-run 模式 | 中 | 无法预览不真正执行 |
| 输出无搜索过滤 | 低 | 大量日志只能滚动查看 |
| 无回滚脚本 | 低 | 只有执行，没有撤销 |

### 优化建议

1. **Cron 定时任务** — 支持周期执行（每天凌晨备份、每周巡检）
2. **任务编排** — 支持串行/并行/条件分支的多步骤 DAG
3. **主机组** — 按服务树/标签动态生成执行目标
4. **脚本编辑器升级** — Monaco Editor（VS Code 同款）+ bash/python 语法高亮
5. **密钥管理** — 加密存储，脚本中用 `${SECRET.db_password}` 引用
6. **执行日志增强** — 支持关键字搜索、按主机过滤、错误行高亮

## 五、Module 05 监控中心

### 当前缺陷

| 问题 | 严重度 | 代码位置 |
|------|--------|---------|
| 告警无去重/聚合 | 高 | CPU>80% 每分钟触发=每分钟一条通知 |
| 无 SLO/错误预算 | 高 | 不知道服务可用性几个 9 |
| 无日志采集/查询 | 高 | 只有指标没有日志 |
| 无 Trace 链路追踪 | 高 | 无法看到请求在微服务间流转 |
| 自定义大盘不可拖拽 | 中 | 布局写死无法个性化 |
| 告警规则无模拟测试 | 中 | 创建规则前不知道会匹配多少 |
| PromQL 无注入防护 | 中 | `monitor_handler.go` 直接透传查询 |
| operator 字段无枚举校验 | 低 | `alert_rule_handler.go:28` 接受任意字符串 |

### 优化建议

1. **告警去重聚合** — 相同指标+相同主机在 N 分钟内只告一次
2. **SLO 管理** — 定义服务目标（99.9%），展示错误预算剩余
3. **日志接入** — Agent 采集日志 → Loki/ES → 平台查询页
4. **可拖拽大盘** — 参考 Grafana，用户自定义布局和图表
5. **规则模拟** — "用过去 24 小时数据测试这条规则会触发几次"
6. **告警升级链** — 5 分钟无人 ack → 通知主管 → 15 分钟无人 → 电话

## 六、Module 06 CI/CD

### 当前缺陷

| 问题 | 严重度 | 代码位置 |
|------|--------|---------|
| 无制品管理 | 高 | 构建产物无存储/版本管理 |
| 无环境隔离策略 | 高 | dev/staging/prod 无区分 |
| 运行日志非实时 | 中 | 无 WebSocket 实时流 |
| Webhook 无签名校验 | 中 | `cicd_handler.go:73-77` 可伪造 |
| 无流水线模板 | 中 | 每条流水线从零配置 |
| 无发布窗口/黑名单 | 中 | 春节期间也能一键发布 |
| 无灰度/金丝雀发布 | 低 | 只有全量发布 |
| 无变量加密 | 低 | Variables 明文存储 |

### 优化建议

1. **制品管理** — 构建产物上传到 S3/MinIO，版本化存储
2. **环境策略** — prod 必须审批、staging 可直接部署、dev 自助
3. **实时日志** — 复用任务中心 WebSocket 能力
4. **发布窗口** — 可配置"周五下午禁止发布"
5. **灰度发布** — 先部署 1 台观察，手动确认后全量
6. **流水线模板** — 预置 Java/Go/Node 标准模板

## 七、通知系统

### 当前缺陷

| 问题 | 严重度 |
|------|--------|
| 无钉钉/飞书/企微推送 | 高 |
| 无通知模板（标题/正文可配） | 中 |
| 无通知频率控制 | 中 |
| 无通知统计（送达率/已读率） | 低 |

### 优化建议

1. **渠道扩展** — 接入飞书/钉钉/企微 Webhook + message-pusher
2. **通知模板** — 按事件类型配置标题和正文模板，支持变量替换
3. **频率控制** — 同一用户同一事件 N 分钟内最多收 1 次
4. **送达看板** — 查看通知的渠道送达率、用户已读率

## 八、全局缺失能力

### 必须补齐

| 能力 | 说明 | 优先级 |
|------|------|--------|
| **Excel 导入导出** | 所有列表页都应支持 | 高 |
| **批量操作** | 资产/工单/任务都需要 | 高 |
| **数据库管理模块** | SQL 工单、慢查询、数据脱敏 | 高 |
| **全局搜索** | 一个搜索框搜资产/工单/任务/告警 | 中 |
| **操作回放** | 关键操作可回放和撤销 | 中 |
| **暗色主题** | 运维值班夜间使用 | 低 |
| **国际化** | 中英文切换 | 低 |
| **移动端适配** | 手机审批工单、查看告警 | 低 |

### 安全加固

| 项目 | 说明 |
|------|------|
| API 全局限流 | 当前仅 login/register 限流 |
| 请求体大小限制 | 防 DoS |
| CORS 白名单 | 当前未配置 |
| JWT 刷新令牌 | 当前单 token 过期后需重新登录 |
| 敏感字段加密 | 云账号密钥、通知配置密码 |
