# BigOps P1 页面点击回归测试用例（稳定化收口）

## 1. 目的

- 验证 P1 稳定化三项能力在真实页面可用：
  - Golden Signals 维度命名映射
  - 巡检报告导出 JSON/CSV
  - 关键监控/巡检页面回归不报阻塞错误

## 2. 测试约束

- 仅允许浏览器页面点击，不允许 API 绕过。
- 使用管理员账号登录（具备监控与巡检权限）。

## 3. 用例清单

### TC-P1-01 监控仪表盘维度表展示可读名称

- 步骤：
  1. 登录后进入 `监控中心 -> 监控大盘`。
  2. 定位 `Golden Signals 维度拆分` 表格。
  3. 检查表头包含：`维度类型`、`维度名称`、`维度键`。
- 预期：
  - 页面可正常渲染，无白屏。
  - 维度类型显示中文（服务/接口/实例/执行人）。
  - 维度名称字段可展示可读值（而非仅技术键）。

### TC-P1-02 巡检记录查看报告

- 步骤：
  1. 进入 `巡检中心`。
  2. 若无数据，先创建模板与计划并点击`立即执行`生成记录。
  3. 在执行记录点击`查看报告`。
- 预期：
  - 报告弹层可打开。
  - 报告 JSON 内容可见且结构完整（record/plan/template/task/status 等字段）。

### TC-P1-03 巡检记录导出 JSON

- 步骤：
  1. 在巡检执行记录行点击`导出JSON`。
- 预期：
  - 浏览器触发下载。
  - 文件名形如 `inspection-record-<id>.json`。

### TC-P1-04 巡检记录导出 CSV

- 步骤：
  1. 在巡检执行记录行点击`导出CSV`。
- 预期：
  - 浏览器触发下载。
  - 文件名形如 `inspection-record-<id>.csv`。
  - CSV 至少包含 `field,value` 表头。
