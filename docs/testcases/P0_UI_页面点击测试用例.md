# BigOps P0 页面点击测试用例（禁止 API 绕过）

## 1. 目的与范围

- 目标：验证 P0 主链路「任务模板 -> 执行列表 -> 执行详情 -> 取消/重试 -> 实时刷新」在真实页面交互下可用。
- 约束：**仅允许通过浏览器页面点击/输入完成测试，不允许 Postman/curl/脚本直接调 API。**
- 覆盖页面：
  - `任务模板页`（`/task/templates`）
  - `执行记录页`（`/task/executions`）
  - `执行详情页`（`/task/executions/:id`）

---

## 2. 测试环境与账号

- 环境：测试环境或本地联调环境（前后端已启动，数据库可写）
- 浏览器：Chrome 最新稳定版
- 账号：管理员账号（具备任务与执行权限）
- 建议准备：
  - 至少 1 台在线 Agent
  - 至少 1 个可执行脚本模板（若无则按用例 01 新建）

---

## 3. 全局通过标准（P0 Exit Criteria）

- 页面无白屏、无阻塞性报错、关键操作可闭环。
- 新建模板后可在列表查到，且字段展示正确。
- 发起执行后，执行记录可见并进入状态流转（`pending/running -> success/failed/canceled`）。
- 执行详情页日志可见，主机行状态与汇总统计一致。
- 取消执行可生效，且状态变为 `canceled`。
- 失败重试与全量重试均可触发新执行记录。
- 自动刷新关闭/开启、刷新间隔切换可生效。

---

## 4. 页面点击测试用例

> 说明：每条用例都按「前置条件 -> 操作步骤 -> 预期结果」执行。

### TC-P0-01 新建任务模板（正向）

- 前置条件：
  - 已登录系统
  - 位于任务中心菜单
- 操作步骤：
  1. 点击左侧菜单进入「任务模板」。
  2. 点击「新建/创建任务模板」按钮。
  3. 输入模板名称：`P0_UI_模板_001`。
  4. 选择脚本类型（bash）。
  5. 输入脚本内容：`echo "P0_UI_OK"`。
  6. 设置超时时间（如 60 秒）。
  7. 点击「保存」。
- 预期结果：
  - 页面提示创建成功。
  - 列表中出现 `P0_UI_模板_001`，状态为启用（或默认状态）。

---

### TC-P0-02 任务模板列表搜索与重置

- 前置条件：
  - 已存在模板 `P0_UI_模板_001`
- 操作步骤：
  1. 在模板页搜索框输入 `P0_UI_模板_001`，点击「搜索/筛选」。
  2. 观察列表结果。
  3. 点击「重置」。
- 预期结果：
  - 搜索后列表仅展示匹配模板。
  - 重置后恢复完整列表。

---

### TC-P0-03 通过页面发起一次任务执行

- 前置条件：
  - 模板 `P0_UI_模板_001` 可见
  - 有可用目标主机（可选在线 Agent 对应 IP）
- 操作步骤：
  1. 在模板行点击「执行」。
  2. 在弹窗/表单中输入目标主机（至少 1 个）。
  3. 点击「确认执行」。
  4. 自动跳转或手动进入「执行记录」页。
- 预期结果：
  - 提示执行已发起。
  - 执行记录列表出现新记录，状态为 `pending` 或 `running`。

---

### TC-P0-04 执行记录页自动刷新与间隔切换

- 前置条件：
  - 执行记录页有至少 1 条运行中或近期记录
- 操作步骤：
  1. 进入「执行记录」页。
  2. 打开「自动刷新」开关。
  3. 切换刷新间隔：`5s -> 10s -> 30s`。
  4. 观察“上次刷新时间/刷新中”状态文案变化。
  5. 关闭自动刷新。
- 预期结果：
  - 间隔切换后，刷新节奏明显变化。
  - 刷新状态文案正常变化，无报错。
  - 关闭后不再自动刷新。

---

### TC-P0-05 进入执行详情并验证实时日志

- 前置条件：
  - 至少存在 1 条可查看详情的执行记录
- 操作步骤：
  1. 在执行记录页点击某条记录「详情」。
  2. 在详情页观察日志区是否持续出现输出。
  3. 点击「暂停滚动/继续滚动」。
  4. 使用日志关键词过滤（输入 `P0_UI_OK`）。
- 预期结果：
  - 日志区有数据或历史日志可见。
  - 暂停/继续生效。
  - 关键词过滤生效，列表缩小到匹配项。

---

### TC-P0-06 取消执行（运行中）

- 前置条件：
  - 存在运行中执行（`running`）
- 操作步骤：
  1. 在执行记录页找到运行中记录。
  2. 点击「取消」并确认。
  3. 观察状态变化（必要时等待自动刷新）。
  4. 进入该记录详情页复核汇总与主机状态。
- 预期结果：
  - 操作提示成功。
  - 列表状态变为 `canceled`。
  - 详情页统计与主机结果一致（未完成主机应被取消）。

---

### TC-P0-07 失败主机重试（scope=failed）

- 前置条件：
  - 有一条包含失败主机的历史执行（`failed/partial_fail/canceled`）
- 操作步骤：
  1. 在执行记录页点击该记录「重试失败」。
  2. 观察提示与新执行记录生成情况。
  3. 进入新执行详情核对目标主机数量。
- 预期结果：
  - 成功创建新执行。
  - 新执行目标仅包含失败/超时/取消主机，不应包含已成功主机。

---

### TC-P0-08 全量重试（scope=all）

- 前置条件：
  - 存在可重试执行记录
- 操作步骤：
  1. 在执行记录页点击「重试全部」。
  2. 打开新执行详情。
  3. 对比原执行总主机数与新执行总主机数。
- 预期结果：
  - 成功创建新执行。
  - 新执行目标主机数与原执行一致（全量）。

---

### TC-P0-09 执行详情统计正确性校验

- 前置条件：
  - 任一已结束执行记录
- 操作步骤：
  1. 进入执行详情页。
  2. 记录汇总卡：总数/成功/失败/成功率/失败率。
  3. 统计主机表中各状态数量。
  4. 对比两者是否一致。
- 预期结果：
  - 汇总卡与主机明细统计一致。
  - 百分比计算合理（四舍五入误差在可接受范围）。

---

### TC-P0-10 权限与异常提示（页面路径）

- 前置条件：
  - 准备一个权限受限账号（无任务执行权限）
- 操作步骤：
  1. 使用受限账号登录。
  2. 进入任务中心尝试执行、取消、重试。
- 预期结果：
  - 页面提示权限不足（或按钮不可见/不可点）。
  - 无越权执行成功情况。
