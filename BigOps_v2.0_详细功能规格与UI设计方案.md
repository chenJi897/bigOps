# BigOps v2.0 详细功能规格与UI设计方案

**文档版本**：v2.0（极详细版）  
**编写日期**：2026年4月9日  
**作者**：AI 编码助手  
**适用对象**：前端开发、后端开发、产品经理、UI设计师、测试人员  
**文档目的**：提供可直接落地执行的极详细功能规格、页面布局、交互逻辑、颜色定义、按钮级别、字段清单等。

## 一、总体设计原则

### 1.1 设计规范
- **主色调**：#409EFF（Element Plus 默认蓝色）
- **成功色**：#67C23A（success）
- **警告色**：#E6A23C（warning）
- **危险色**：#F56C6C（danger）
- **信息色**：#909399（info）
- **按钮级别规范**：
  - Primary（主要按钮）：type="primary"，蓝色，重要操作
  - Success：type="success"，绿色，用于“完成”“执行成功”相关
  - Danger：type="danger"，红色，用于“删除”“停止”“强制终止”
  - Warning：type="warning"，橙色，用于“重试”“暂停”
  - Default：默认按钮，灰色，用于“取消”“详情”“查看”

### 1.2 状态颜色统一规范
- **Pending（待执行）**：#909399（灰色）
- **Running（执行中）**：#409EFF（蓝色）
- **Success（成功）**：#67C23A（绿色）
- **Failed（失败）**：#F56C6C（红色）
- **Timeout（超时）**：#E6A23C（橙色）
- **Canceled（已取消）**：#606266（深灰）

## 二、阶段一核心模块详细规格

### 2.1 统一任务中心（最高优先级）

#### 2.1.1 任务列表页（/task-center）

**页面布局**：
- 顶部：搜索栏 + 筛选条件栏 + 操作按钮区
- 左侧：可选的服务树筛选（可折叠）
- 中间：任务列表表格（支持分页）
- 右侧：可选的任务统计卡片（今日执行、成功率、平均耗时）

**顶部筛选条件**（从左到右）：
- 搜索框：支持任务名称、任务ID、执行人模糊搜索（宽度 320px）
- 下拉选择：任务类型（全部、脚本执行、巡检任务、CI/CD、工单自动化）
- 下拉选择：执行状态（全部、待执行、执行中、成功、失败、超时）
- 日期范围选择器：执行时间范围
- Button（primary）：“新建任务” —— 蓝色主按钮，置于最右侧

**表格列定义**（从左到右）：
1. 任务ID（可点击跳转详情）
2. 任务名称（支持换行，最宽 240px）
3. 任务类型（Tag 标签显示）
4. 执行状态（使用上述状态颜色 + ElTag）
5. 执行人（用户名）
6. 执行节点数（例如 12/15）
7. 成功率（进度条 + 百分比，绿色/红色区分）
8. 开始时间
9. 耗时（单位：秒或分钟）
10. 操作列（宽度 180px）

**操作列按钮定义**（从左到右）：
- “执行”按钮：type="primary"，蓝色，仅在 Pending 状态显示
- “查看日志”按钮：type="default"，灰色，始终显示
- “重试”按钮：type="warning"，橙色，仅在 Failed/Timeout 状态显示
- “更多”下拉菜单（包含：编辑、停止、删除、查看报告）

**统计卡片颜色**：
- 今日执行次数：蓝色卡片
- 成功率：绿色卡片（>90%）/橙色卡片（70-90%）/红色卡片（<70%）
- 失败任务数：红色卡片

**权限控制**：
- 普通用户只能看到自己创建和被授权的任务
- 管理员可看到全部任务
- “新建任务”按钮只有拥有 `task:create` 权限的用户可见

#### 2.1.2 任务详情页

**页面结构**：
- 顶部面包屑 + 任务基本信息卡片
- Tab 切换：基本信息 | 执行记录 | 实时日志 | 执行报告 | 关联工单

**实时日志区域设计**：
- 顶部工具栏：自动刷新开关（默认开启）、暂停按钮（danger）、清空按钮（default）
- 日志展示区：黑色背景，白色+绿色+红色文字（区分 stdout/stderr）
- 底部状态栏：当前执行状态、耗时、进度百分比

**执行记录表格**：
- 节点IP、执行状态（颜色标签）、开始时间、结束时间、耗时、退出码
- 操作按钮：仅“查看日志”（蓝色链接）

### 2.2 智能告警中心（第二优先级，重点详细）

#### 2.2.1 告警事件列表页（/alert-events）

**参考设计图**：alert-events-v4.png、alert-events-table-v2.png、alert-events-final.png

**页面布局**：
- 顶部：Tab 切换（全部告警 | 未处理 | 已认领 | 已解决 | 已关闭）
- 筛选栏（一行）：时间范围 + 告警级别 + 服务树 + 负责人 + 告警源
- 视图切换按钮组（位于筛选栏右侧）：
  - “表格视图”按钮（default）
  - “时间轴视图”按钮（default）
  - “统计视图”按钮（default）

**表格视图列定义**（推荐默认视图）：
1. 告警ID（蓝色可点击）
2. 告警标题（支持高亮搜索关键词）
3. 告警级别（Tag：P0红色、P1橙色、P2蓝色、P3灰色）
4. 所属服务树（可折叠显示）
5. 负责人（头像+姓名）
6. 首次发生时间
7. 持续时长（动态计算，颜色：>1h 红色，>30min 橙色）
8. 当前状态（Tag：未处理红色、已认领橙色、已解决绿色）
9. 操作列（宽度 220px）

**操作列按钮详细定义**：
- **主要操作按钮组**（ElButtonGroup）：
  - “认领”按钮：type="primary"，蓝色，图标为用户图标
  - “解决”按钮：type="success"，绿色，图标为对勾
  - “转工单”按钮：type="warning"，橙色，图标为文档
- “更多”按钮（default）：下拉菜单包含：抑制告警、查看详情、关联知识、删除事件

**时间轴视图**：
- 左侧时间轴竖线（蓝色）
- 每个事件卡片包含：时间、标题、级别标签、简要描述、操作按钮（同上）

**批量操作栏**（当选中多条时在表格上方出现）：
- 按钮组从左到右：
  - “批量认领”（primary，蓝色）
  - “批量解决”（success，绿色）
  - “批量转工单”（warning，橙色）
  - “批量抑制”（default，灰色）
  - “取消选择”（default）

**颜色规范**：
- P0 告警：#F56C6C（深红）
- P1 告警：#E6A23C（橙色）
- P2 告警：#409EFF（蓝色）
- 未处理状态背景：浅红色 (#FEF0F0)
- 已解决状态背景：浅绿色 (#F0F9EB)

#### 2.2.2 告警详情页

**页面结构**：
- 左侧：告警基本信息 + 时间线（垂直时间轴）
- 右侧：关联信息（关联资产、关联知识、关联工单、历史相似告警）

**时间线节点颜色**：
- 告警产生：红色圆点
- 认领：蓝色圆点
- 处理过程：橙色圆点
- 解决：绿色圆点

**操作按钮区**（固定在右上角）：
- “立即认领”（primary）
- “标记解决”（success）
- “创建工单”（warning）
- “关联知识”（default）

## 三、技术实现注意事项

1. 所有列表页必须支持 URL 参数保存筛选状态（刷新后状态保留）
2. 实时日志必须使用 WebSocket，长连接断开后自动重连
3. 所有危险操作（删除、强制停止）必须二次确认弹窗
4. 重要按钮必须有 loading 状态防止重复点击
5. 所有颜色必须严格遵循上述状态色和按钮色规范
